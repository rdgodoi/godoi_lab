Guia GIT



git add -A: Prepara todas as alterações para que possam ser comitadas


git status: Verifica de há alterações, inserções ou exclusões a serem feitas comparado ao ultimo commit


git commit -m "Mensagem": Envia as alterações para a branch atual


git log: Exibe o histórico de commit


git branch: verificar qual a branch que estamos trabalhando

git commit -am "Mensagem": faz o commit já adicionando as alterações gerais

git diff: exibe quais alterações foram feitas depois do ultimo commit de maneira mais detalhada

git diff --name-only: exibe apenas os nomes dos arquivos alterados



Git RESET:

git reset --soft {id}: Volta o estado anterior mantendo as alterações feitas sem serem comitadas

git reset --mixed {id}: Volta o estado anterior mantendo as alterações feitas porem sem prepara-las para o novo commit

git reset --hard {id}: Votla o estado anterior apagando todas as alterações que foram feitas

Git BRANCHS

git branch {teste}: Cria um novo Branch com o nome "teste" copiando o estado da branch atual

git checkout {teste}: Altera o workarea para o branch teste
